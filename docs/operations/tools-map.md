# Mapa de herramientas · Studio Nomade

Cómo se relaciona Noma con el stack operativo del estudio. Noma es la **fuente de verdad**
comercial; complementa (no reemplaza) las herramientas de producción.

| Herramienta             | Para qué                                                     | Relación con Noma                               |
| ----------------------- | ------------------------------------------------------------ | ----------------------------------------------- |
| **Noma**                | Clientes, proyectos, briefs, propuestas, servicios, pipeline | Núcleo operativo/comercial                      |
| **Google Drive**        | Archivos por área → cliente → proyecto (carpetas fechadas)   | Link por cliente/proyecto; integración futura   |
| **Gemini / Meet Notes** | Minutas de reuniones (en Drive)                              | Link a minuta; vínculo futuro                   |
| **Google Calendar**     | Agenda de reuniones                                          | Futuro: próximas reuniones por proyecto         |
| **Google Meet**         | Reuniones cliente/equipo                                     | Guardar link de reunión                         |
| **Slack**               | Comunicación interna (canales por cliente/proyecto/área)     | Link a canal; alertas futuras                   |
| **Asana**               | Control y avance de proyectos (réplica por áreas)            | Link a proyecto; avance futuro (portal cliente) |
| **Canva**               | Presentaciones y piezas editables                            | Link a presentación                             |
| **Figma**               | Diseño                                                       | Link en proyecto                                |

## Organización en Drive (convención)

`Unidad de área / Cliente / Proyecto (fechado) / ...`

Noma documenta estos enlaces vía `resource_links` para centralizar el acceso y preparar la
automatización. Ver [integrations.md](../technical/integrations.md).
